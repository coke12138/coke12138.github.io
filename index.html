DNS配置流程

全流程

在Centos7上
主DNS服务器，不带chroot
1. ifconfig 查看IP（固定IP），并把DNS指向自己
vim /etc/sysconfig/network-scripts/ifcfg-ens33
修改保存退出。
2.检查软件安装
yum -y install bind
3.配置主配置文件
vim /etc/named.conf
修改两个any：
options {
listen-on port 53 { 127.0.0.1; };
allow-query { localhost; };
};
把127.0.0.1和localhost都改成any，其余不动。
保存退出。
4.新增正反向区
vim /etc/named.rfc1912.zones
新增正向和反向解析区

保存退出。
5.创建并且配置正向数据库文件
cp /var/named/named.localhost /var/named/jnrplinux.zone
vim /var/named/jnrplinux.zone

保存退出。

6.创建并且配置反向数据库文件
cp /var/named/named.localhost /var/named/jnrplinux.rev.zone
vim /var/named/jnrplinux.rev.zone

保存退出。

7.修改权限或者更改属主,可用ll查看文件权限
cd /var/named/
chmod 644 /var/named/jnrplinux.zone
chmod 644 /var/named/jnrplinux.rev.zone
或
chown root:named jnrplinux.zone
chown root:named jnrplinux.rev.zone

8.检查dns语法
named-checkconf //检查主配置文件
named-checkzone jnrplinux.com jnrplinux.zone
named-checkzone 118.168.192.in-addr.arpa jnrplinux.rev.zone
9.启动服务并查看状态
systemctl start named.service
systemctl status named.service
关闭ipv6解析（非必须，可跳过，进入下一步）
vi /etc/sysconfig/named
把下面的代码插入文件末尾
OPTIONS=”-4”
保存退出。
重启服务并查看状态
systemctl restart named.service
systemctl status named.service
10.本地lookup测试
nslookup jnrplinux.com
nslookup 192.168.118.151
11.暂时关闭防火墙 或 添加规则并重启防火墙
systemctl stop firewalld.service
或
firewall-cmd –add-service=dns –permanent
firewall-cmd –reload
12.客户机修改dns指向本服务器并进行nslookup测试

从DNS服务器搭建（辅助DNS服务器）
在另一台Centos7上
13.ifconfig 查看IP（固定IP），并把DNS指向自己
vim /etc/sysconfig/network-scripts/ifcfg-ens33
修改保存退出。

19.启动服务并查看状态
systemctl start named.service
systemctl status named.service
关闭ipv6解析（非必须，可跳过，进入下一步）

保存退出。

20.本地nslookup测试

下面是带chroot的流程：
在一个Cnetos7上
1. ifconfig 查看IP（固定IP），并把DNS指向自己
vim /etc/sysconfig/network-scripts/ifcfg-ens33
修改保存退出。
2.检查软件安装
yum install bind-chroot bind -y
3.复制生成文件

4.编辑主配置文件
vim /var/named/chroot/etc/named.conf
修改两个any，保存退出。
5.新增正反向区
cp /etc/named.rfc1912.zones /var/named/chroot/etc/
chmod 646 /var/named/chroot/etc/named.rfc1912.zones
vim /var/named/chroot/etc/named.rfc1912.zones
新增正向和反向解析区

保存退出。
6.创建并且配置正向数据库文件

vim /var/named/chroot/var/named/kjxy.zone

保存退出。

7.创建并且配置反向数据库文件

vim /var/named/chroot/var/named/kjxy.rev.zone

保存退出。
修改权限或者更改属主,可用ll查看文件权限
cd /var/named/chroot/var/named/
chmod 644 kjxy.zone
chmod 644 kjxy.rev.zone
8.检查dns语法
named-checkconf -z /var/named/chroot/etc/named.conf
named-checkconf /var/named/chroot/etc/named.conf
named-checkzone kjxy.net /var/named/chroot/var/named/kjxy.zone
named-checkzone 118.168.192.in-addr.arpa /var/named/chroot/var/named/kjxy.rev.zone
9.启动服务并查看状态
systemctl start named-chroot
systemctl status named-chroot
10.本地lookup测试
nslookup kjxy.net
nslookup 192.168.118.151
11.暂时关闭防火墙 或 添加规则并重启防火墙
systemctl stop firewalld.service
12.客户机修改dns指向本服务器并进行nslookup测试

搭建个人博客

搭建个人博客-1

(在win10上–静态博客–Hexo–GitHub)

1.安装node.js

官方：https://nodejs.org/en/
腾讯软件（版本不是最新的哦！）（选普通下载！）： https://pc.qq.com/search.html#!keyword=node
安装就一直下一步，有一个地方的勾选我选上了。
验证，cmd下成功查看node -v 和 npm -v 。
我此时的版本node v12.18.4，npm 6.14.6。

2.用npm在淘宝源下载cnpm（特色）

命令： npm install -g cnpm –registry=https://registry.npm.taobao.org （希望没敲错）
成功的话在命令行将看到进度条和显示一系列的自动操作。我用了二十多秒。
验证，查看cnpm版本：6.1.1
(这一步据说可以用yarn完成）

3.下载Hexo

命令：cnpm install -g hexo-cli
回车，命令行跑一段。
验证，查看hexo -v 这里是4.2.0
同时也可以看到系统和其余组件版本。

4.新建文件夹！

没错，为博客在本地的文件建立一个新文件夹。
cmd进入此文件夹操作。

5.hexo初始化

命令：hexo init
(十分钟后）

6.启动博客

hexo s
localhost:4000 可访问预览博客
ctrl + C 中断

7.写文章

文章存储路径：.\source_posts
markdown格式保存
用一个本文编辑器以markdown格式写一篇文章放进去即可。

在刚才的新建文件夹下执行命令：
hexo clean 清理
hexo g 生成
hexo s 再次启动查看

8.在GitHub上创建仓库

(你必须先有GitHub账号)
新仓库名称格式：GitHub昵称.github.io
公共仓库

9.在新文件夹中安装git

在刚才的新建文件夹下执行命令：
cnpm install –save hexo-deployer-git

10.设置 _config.yml文件

编辑文件内容：
在末尾的

改成：（冒号后有空格,字符用单引号括起来,仅windows）

11.部署到远端

在刚才的新建文件夹下执行命令：hexo d
验证：可以通过仓库地址“XXX.github.io”访问博客。
